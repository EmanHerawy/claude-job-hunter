# Claude Job Hunter

AI-powered job search toolkit built as Claude Code slash commands.

## Project Structure

```
commands/           # Slash command skill files (.md) — each is a self-contained skill prompt
resources/          # Research checklists and reference templates used by skills
examples/           # Example outputs for skills to reference
profile/            # candidate-profile.md template (user profiles are gitignored)
```

## Key Files

- `commands/setup.md` — Bootstrap installer; symlinks all skills into `~/.claude/commands/`
- `commands/uninstall.md` — Removes symlinks and config
- `resources/research-checklist.md` — Company research structure used by multiple skills
- `resources/interview-research-checklist.md` — Leadership/culture research structure
- `profile/candidate-profile.md` — Template for user profiles (tracked in git)

## Conventions

### Skill file structure
Each skill in `commands/` follows this pattern:
1. **Title and description** — what the skill does
2. **Inputs** — `$ARGUMENTS` parsing with quoted strings, required/optional markers
3. **Setup** — Path resolution: config file → env var → conventional paths
4. **Profile Resolution** — Multi-profile support (explicit arg → config default → auto-detect)
5. **Existing Context** — Check for prior outputs from other skills
6. **Output** — Directory and file structure for results
7. **Workflow** — Phased execution with user confirmation between phases
8. **Quality Checklist** — Verification items before finishing

### Path resolution (all skills)
1. Read `~/.claude/.claude-job-hunter.conf` for configured paths
2. Fall back: `$CLAUDE_JOB_HUNTER_DIR` → `./claude-job-hunter/` → `~/claude-job-hunter/`

### Config file
`~/.claude/.claude-job-hunter.conf` — generated by `/setup`, contains:
- `REPO_DIR`, `WORK_DIR`, `PROFILE_DIR`, `EVALUATIONS_DIR`, `CARDS_DIR`
- `RESOURCES_DIR`, `EXAMPLES_DIR`, `WATCHLIST_FILE`, `SEARCH_LOG_FILE`
- `ACTIVE_PROFILE` — default profile name

### Output directories
- `evaluations/[company]-[role-slug]/` — per-role evaluation outputs
- `evaluations/[company]/` — company-level outputs (e.g., SWOT analysis)
- `cards/[company]-[year]/` — player card deployments
- `profile/` — user profiles (gitignored except template)

### Adding a new skill
1. Create `commands/[skill-name].md` following the structure above
2. Add it to `commands/setup.md` install list and summary display
3. Update `README.md` — description list, usage examples, command docs, repo structure, lifecycle

## Development Notes

- Skills are pure markdown prompts — no code to build or test
- User data (profiles, evaluations, cards) is gitignored; only templates and skills are tracked
- Skills use `WebSearch` and `WebFetch` for live research
- Skills reuse each other's outputs (e.g., `/offer-negotiation` reads `COMP_RESEARCH.md`)
- All research findings must include source URLs — no unsourced claims
