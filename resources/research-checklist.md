# Company Research Checklist

Use WebSearch and WebFetch to research each area. Capture findings with source URLs for the COMPANY_RESEARCH.md output file.

## Required Research Areas

### 1. Company Basics
- [ ] Official website — about page, mission statement
- [ ] Wikipedia or Crunchbase profile
- [ ] Founding year, founders, headquarters
- [ ] Employee count / team size
- [ ] Funding history (stage, amounts, investors)

### 2. Product & Business Model
- [ ] What do they sell/build?
- [ ] Revenue model (SaaS, usage-based, enterprise, etc.)
- [ ] Target customers (B2B, B2C, developer-focused, enterprise)
- [ ] Key integrations or partnerships
- [ ] Pricing page (signals about customer tiers)

### 3. Technical Infrastructure
- [ ] Engineering blog posts (search "[company] engineering blog")
- [ ] Tech stack mentions in job postings
- [ ] Open source projects or contributions
- [ ] Cloud provider(s) used
- [ ] API documentation (if applicable)

### 4. Security Context
- [ ] Existing security team (check LinkedIn, job postings)
- [ ] Compliance certifications mentioned (SOC2, ISO, HIPAA, etc.)
- [ ] Security-related blog posts or announcements
- [ ] Bug bounty program
- [ ] Trust/security page on website
- [ ] Data handling and privacy policies

### 5. Culture & Values
- [ ] Company values page
- [ ] Glassdoor reviews (overall rating, culture themes)
- [ ] Engineering culture signals (remote-friendly, async, etc.)
- [ ] Leadership team backgrounds
- [ ] Recent press about culture or workplace

### 6. Recent News & Context
- [ ] Last 6 months of news coverage
- [ ] Recent product launches or announcements
- [ ] Any security incidents or breaches
- [ ] Hiring patterns (growing, contracting, new areas)
- [ ] Competitor landscape and market position

## Suggested Search Queries

For each company, run these searches (replace [COMPANY]):
1. `"[COMPANY]" about mission values`
2. `"[COMPANY]" engineering blog technical`
3. `"[COMPANY]" security team compliance SOC2`
4. `"[COMPANY]" funding series investors crunchbase`
5. `"[COMPANY]" glassdoor reviews culture`
6. `"[COMPANY]" news 2026`
7. `"[COMPANY]" tech stack infrastructure`
8. `"[COMPANY]" careers security engineer`
9. `site:[company-domain] security OR trust OR compliance`
10. `site:[company-domain] blog engineering`
